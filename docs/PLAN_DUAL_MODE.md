# 双模式解读升级计划

## 背景

当前 translator.py 对所有推文使用同一个 prompt，无差别处理。实际上推文信号价值差异很大：
- Anthropic 发个功能更新 → 只需讲清楚 + 点出亮点（轻模式）
- Sam Altman 在敏感时间点发了一条暧昧推文 → 需要分析信号、动机、时机（深模式）

本次升级核心：**双模式解读 + 写作风格重写**，在现有 x2xhs 架构上演进，不建新项目。

---

## 写作目标

1. 这不是新闻项目，而是 X 上关键人物高信号推文的中文解读项目
2. 目标不是复述或翻译，而是帮助中文读者多看懂一层
3. 长期目标是建立稳定可信、有判断力的个人内容 IP
4. 核心竞争力：稳定可信的判断力 + 自然有辨识度的表达

---

## 双模式设计

### 轻解读（light）

**触发条件：** 功能发布、产品更新、API 公告、常规分享、研究论文、活动通知、数据播报

**任务：** 让中文读者清晰理解这条推文在说什么，并点出为什么值得关注

**结构：**
1. 简明交代发布者、时间和核心内容
2. 提炼 2–4 个关键事实或数据
3. 一句话收尾：为什么这条值得看

### 深解读（deep）

**触发条件：** 敏感时机的暧昧措辞、创始人/CEO 战略表态、竞争信号、舆论管理、监管回应

**任务：** 帮中文读者「多看懂一层」，揭示字面背后的信号与含义

**四层分析框架：**
1. 字面内容：他说了什么
2. 真实含义：他真正想表达什么
3. 行为目的：为什么在此时说这个
4. 行业位置：这对行业格局意味着什么

**信号分类：** 产品信号 / 竞争信号 / 舆论管理 / 预期管理 / 姿态表态 / 市场试探 / 监管回应 / 品牌叙事 / 创始人风格输出

---

## 写作风格规范

- **像谁：** 像一个敏锐、自然、讲得明白的人在解释一条值得看的推文
- **语言：** 干净、有判断力，不啰嗦、不堆砌形容词
- **结尾：** 落在「干净判断」上，不做互动引导
- **严禁：**
  - 官媒腔：「高度重视」「积极探索」「深度融合」
  - 营销号标题党：「震惊！」「颠覆！」「重磅！」「不可思议！」
  - 结尾互动：「你觉得如何呢」「快来评论」「一起来看看」
  - URL 和 @用户名
- **深模式分析原则：** 推测要明确标注「可能」「或许」，不靠臆测，保持克制

---

## 实现方案

### 改动范围（当前阶段：小红书）

| 文件 | 改动 |
|------|------|
| `processor/prompts.py` | 新建，集中管理 prompt（双模式 system prompt） |
| `processor/translator.py` | 替换 `_system_prompt()`，TranslationPayload 增加 `mode` 字段 |
| `scraper/models.py` | ProcessedContent 增加 `mode: str` 字段 |
| `storage/database.py` | 添加 migration：`mode` 列 |
| `storage/tweet_repo.py` | `save_sent()` 写入 `mode` |

### 不改动

- `scraper/` — 抓取逻辑不变
- `publisher/image_overlay.py` — 图片叠加不变
- `pipeline.py` — deliver 流程不变
- `bot.py` — Telegram bot 不变

### 路由设计

**不使用独立分类器 LLM call。** 让 LLM 在主翻译的同一次 call 里先判断信号等级，再按对应模式生成。省一次 API call，降低延迟和成本。

## 多平台扩展（待实现）

当前只做小红书。后续扩展时在 `processor/prompts.py` 增加 `platform_adapter(platform)` 函数：

| 平台 | 篇幅 | 结构 |
|------|------|------|
| 小红书 | 200–800 字 | 短段落，克制使用 emoji |
| 微信公众号 | 800–2000 字 | 小标题分节，可有引用块 |
| 视频文案 | 60–180 秒口播稿 | 开头 hook → 主体 → 结尾判断 |

---

## 验证方式

1. 选 10 条真实推文（5 条低信号 + 5 条高信号）
2. 用新 prompt 跑一遍，对比旧输出
3. 人工按「写作风格规范」逐条评分
4. 确认后部署到 VPS
